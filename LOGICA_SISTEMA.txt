===============================================================================
                    SISTEMA DE GESTIÓN DE INVENTARIO DE BIENES
                          LÓGICA Y ESTRUCTURA DEL SISTEMA
===============================================================================

1. ORGANISMOS
   ────────────────────────────────────────────────────────────────────────
   - Entidad superior que agrupa todas las instituciones
   - Ejemplo: MINISTERIO DEL PODER POPULAR PARA LA EDUCACIÓN UNIVERSITARIA
   
   Atributos:
   • Código: Identificador único del organismo
   • Nombre: Denominación de la institución
   
   Relación: Un organismo tiene muchas Unidades Administradoras


2. UNIDADES ADMINISTRADORAS
   ────────────────────────────────────────────────────────────────────────
   - Subdivisions dentro de un organismo encargadas de administrar áreas específicas
   - Ejemplo: UPTOS "CLODOSBALDO RUSSIAN"
   
   Atributos:
   • Código: Identificador único (1430)
   • Denominación: Nombre de la unidad (OFICINA DE ADMINISTRACIÓN)
   • Organismo: Relación con el organismo padre
   
   Relación: Una unidad administradora tiene muchas Dependencias


3. DEPENDENCIAS
   ────────────────────────────────────────────────────────────────────────
   - Departamentos o áreas dentro de una unidad administradora
   - Ejemplo: DEPENDENCIA USUARIA
   
   Atributos:
   • Código: Identificador único (0)
   • Denominación: Nombre del departamento
   • Unidad Administradora: Relación con la unidad padre
   
   Relación: Una dependencia tiene muchos Bienes y muchos Usuarios asignados


4. BIENES
   ────────────────────────────────────────────────────────────────────────
   - Artículos o activos físicos que pertenecen a una dependencia
   - Son registrados, almacenados y rastreados por el sistema
   
   Atributos:
   • Código: Identificador único del bien
   • Descripción: Detalles del artículo (ej: computadora, escritorio, etc.)
   • Precio: Valor monetario del bien
   • Dependencia: Unidad a la que pertenece
   • Responsable: Usuario responsable del cuidado
   • Ubicación: Dónde se encuentra el bien
   • Fecha de Registro: Cuándo se incorporó al inventario
   • Estado: Activo, Inactivo, En Mantenimiento, etc.
   • Fotos: Evidencia visual del bien
   
   Relación: 
   - Un bien pertenece a una Dependencia
   - Un bien es responsabilidad de un Usuario (Responsable)
   - Un bien puede tener muchos Movimientos


5. USUARIOS
   ────────────────────────────────────────────────────────────────────────
   - Personal responsable de administrar los bienes y el sistema
   
   Atributos:
   • Cédula: Identificación nacional
   • Nombre: Nombre completo
   • Correo: Email de contacto
   • Contraseña: Acceso al sistema
   • Rol: Tipo de usuario (Administrador, Gerente, Responsable)
   • Activo: Estado del usuario en el sistema
   
   Tipos de Usuarios:
   - Administrador: Gestión completa del sistema
   - Gerente de Bienes: Supervisa inventarios
   - Usuario Responsable: Cuida bienes asignados
   
   Relación:
   - Un usuario puede ser responsable de muchos Bienes
   - Un usuario genera movimientos de bienes


===============================================================================
                           FLUJO DE OPERACIÓN
===============================================================================

1. REGISTRO INICIAL:
   • Se crea un Organismo (institución)
   • Se crean Unidades Administradoras dentro del organismo
   • Se crean Dependencias dentro de cada unidad
   • Se registran Usuarios con sus respectivos roles

2. INGRESO DE BIENES:
   • Se asigna cada bien a una Dependencia
   • Se asigna un Usuario responsable de cada bien
   • Se registran detalles: código, descripción, precio, ubicación, fecha

3. ADMINISTRACIÓN:
   • Se rastrean cambios de ubicación (Movimientos)
   • Se registran responsables por cambios (Usuarios)
   • Se mantiene historial de todas las transacciones

4. CONSULTAS Y REPORTES:
   • Se pueden generar reportes por Dependencia
   • Se pueden filtrar bienes por Organismo
   • Se puede ver historial de movimientos por bien


===============================================================================
                           RELACIONES PRINCIPALES
===============================================================================

Organismo
    ↓ (1 a muchos)
Unidad Administradora
    ↓ (1 a muchos)
Dependencia
    ↓ (1 a muchos)
Bienes
    ↓ (relación con)
Usuario (Responsable)


===============================================================================
                           RESPONSABILIDADES
================================================================================

RESPONSABLE PATRIMONIAL PRIMARIO:
- Supervisa el inventario completo
- Valida registros de bienes
- Genera reportes maestros
- Audita el sistema

RESPONSABLE PATRIMONIAL POR USO:
- Cuida bienes específicos asignados
- Reporta cambios de ubicación
- Mantiene bienes en buen estado
- Registra daños o movimientos

===============================================================================
                           NOTAS IMPORTANTES
================================================================================

• El sistema es jerárquico: Organismo > Unidad > Dependencia > Bien
• Cada nivel inferior pertenece únicamente a uno de nivel superior
• Los bienes son la entidad central que se rastrea
• Los usuarios son responsables de los cambios en bienes
• Se mantiene un historial completo de movimientos y cambios
• El sistema genera reportes por dependencia para auditoría
